Wanna TEMPEST your computer?

Étape 4

Résumons les étapes précédentes :

  • Nous avons vu qu'un ordinateur émettait des ondes électromagnétiques;
  • Nous avons écrit un programme permettant de transmettre une séquence binaire constituée d'une alternance de 0 et de 1 à destination de la RAM;
  • Nous savons réceptionner le signal émis par la RAM dans un fichier sur l'ordinateur de l'attaquant.

À présent, revenons à l'objectif principal de ce challenge : transmettre une information (un flag constitué d'une chaîne de caractère) contenu dans un fichier de l'ordinateur de la victime. Pour cela, nous pourrons stimuler la RAM comme dans l'étape 2 et réceptionner l'information comme dans l'étape 3. Nous pourrons enfin afficher le résultat sous forme d'une chaîne de caractère sur l'ordinateur de l'attaquant.

Il reste donc deux étapes à réaliser :

  • La transmission : écrire le malware permettant transmettre le flag en continu;
  • La réception : écrire le programme affichant le binaire reçu sous forme de chaîne de caractère.


Transmission

À présent, il s'agit de modifier notre script de transmission afin qu'il émette le binaire associé à un fichier précis.

Cette étape étant trop longue pour être réalisée dans le temps imparti, vous pouvez télécharger ici le fichier C modifié. Celui-ci prend un fichier en argument, lit son code binaire, écrit dans la RAM pendant 1 seconde pour transmettre un bit 1 et ne fait rien pendant 1 seconde pour transmettre un bit 0. Le temps de transmission d'un bit est ici fixé à 1 seconde afin de limiter le taux d'erreur à la réception.

Le programme tourne en boucle à l'infini et, avant de transmettre le binaire, transmet un pattern qui ne correspond à aucun caractère ASCII. Ce pattern permet de localiser la position du flag lors de la réception : celui-ci se trouvera ainsi entre deux occurrences du pattern. Nous avons choisi pattern='11111111'.

Exécuter alors le malware sur l'ordinateur de la victime.

Passons à présent à l'étape de réception.


Reception

Nous allons maintenant construire le script C qui va lire le message reçu et afficher le flag sous forme d'une chaîne de caractère.

Voici la fonction qui permet d'extraire le message entre deux occurrences d'un pattern :

  void extract_msg(FILE * fic, char * msg, char * pattern)
  {
    // Extrait la charge utile du fichier obtenu à la réception du signal
    // Met le résultat en binaire dans la chaine "msg"
    char chaine[TAILLE_MAX],
         tmp1[TAILLE_MAX], // Les tmp sont des chaines temporaires pour le formatage.
         *tmp2,
         tmp3[TAILLE_MAX]="",
         *tmp4,
         bin[TAILLE_MAX]="";

    while (fgets(chaine, TAILLE_MAX, fic) != NULL) // On lit chaque ligne du fichier
                                                   // dans chaine.
    {
      memset (tmp1, 0, sizeof (tmp1)); // On initialise tmp1 à la chaine vide.
      strncpy(tmp1, chaine+10, 55); // On garde garde la partie binaire de chaine
                                    // dans tmp1.
      rem_space(tmp1); // On enlève les espaces de tmp1.
      strcat(bin, tmp1); // On récupère le binaire entier du fichier dans bin.
    }
    tmp2 = strstr(bin, pattern); // On stocke la position de la première occurrence
                                 // du pattern dans tmp2.
    strcpy(tmp3, tmp2+strlen(pattern)); // On enlève ce pattern et on stocke le
                                        // résultat dans tmp3.
    tmp4 = strstr(tmp3, pattern); // On localise la deuxième occurrence du pattern.
    strncpy(msg, tmp3, tmp4-tmp3); // On garde uniquement le binaire du flag dans msg.
  }
